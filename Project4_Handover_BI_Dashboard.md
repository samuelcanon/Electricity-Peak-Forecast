## SHARED MISSION (applies to every project in this portfolio)
This portfolio is three analysis projects (all strong, being re-framed to read as strong) plus this BI dashboard, targeting entry-level data analyst / data scientist roles (Netherlands + Switzerland).

**Why Project 4 exists:** all three existing projects are notebooks. None is a dashboard; none uses a BI tool; the visible-insight axis is the portfolio's thinner side. Real Netherlands job postings (searched this session) repeatedly ask for turning data into **actionable insights**, communicating **recommendations to management**, and name **Power BI / Tableau / Looker**. Project 4 fills that gap with ONE clear, positive, visually communicated, stakeholder-facing insight.

**Non-negotiable principles (all projects):**
- Defensibility is the ceiling. Findings reported straight, tagged (certain)/(likely)/(guessing).
- No citations from memory. No overclaiming; state limitations.
- Subtraction over addition — keep it SMALL.

---

## WHAT PROJECT 4 IS (and is NOT)
- **Is:** a small, decision-focused BI dashboard built around a DECISION from the start — 3–5 linked visuals, one honest headline insight a stakeholder would act on.
- **Is NOT:** another Project 3. This is a weekend-sized project. Scope creep is the enemy.
- **Is NOT** a completion trophy: it raises the portfolio ceiling and adds the BI/communication axis, but readiness = cold-defensibility of ALL projects, not a project count. Portfolios are SAMPLED not averaged — a reviewer opens one or two projects; this one raises the odds a strong one gets seen, it doesn't "balance" the others.

## HOW TO SCOPE IT (once tool is chosen)
1. **One dataset** — clean, legible, ideally in a domain that complements the portfolio (energy would tie to Project 3 and to the electricity-market role seen in postings; not required).
2. **One decision-relevant question** — phrased as "which X should [stakeholder] prioritise, and what does the data say to do?"
3. **3–5 linked visuals** — each earning its place; no clutter.
4. **One honest headline insight** — stated as a recommendation, with its limitation.
5. **Same discipline** as the notebooks: defensible, tagged, limitations stated.

## WHAT MAKES IT SPECIAL (to build in, for study guide + interview)
- It's the project that demonstrates **communication and BI-tool fluency** — the axis the three notebooks don't cover.
- It's built around a **decision**, so its output is actionable by construction (unlike the analyses, which were scoped as investigations).
- Interview line to aim for: "Projects 1–3 show I can validate, test, and model honestly; Project 4 shows I can turn data into a decision a stakeholder acts on, in the tool they actually use."
